# Antigravity + Claude Code: Multi-Agent 開發架構

## 1. 架構概述 (Architecture Overview)

這是一個結合 **Antigravity (多 Agent 協作平台)** 與 **Claude Code CLI (單一編碼 Agent)** 的強大開發架構。
在這個架構中，Antigravity 扮演 **「總指揮 (Orchestrator) 與系統整合者」**，而 Claude Code 扮演 **「專業編碼員 (Specialist Coder)」**。

### 角色分配
- **Antigravity (主控 Agent)**：
  - 負責理解使用者的宏觀需求，進行系統分析與架構設計。
  - 將大任務拆解成多個子模組 (Sub-tasks)。
  - 管理跨專案的上下文、讀取知識庫 (Vault/KIs)、操作瀏覽器測試。
  - 透過終端機 (Terminal) 指令呼叫/排程 Claude Code 進行具體開發。
  - 審查 Claude Code 產出的程式碼，並進行最終的整合與測試。

- **Claude Code CLI (執行 Agent)**：
  - 專注於單一資料夾或模組內的程式碼修改與生成。
  - 具備深入理解 Codebase 的能力，自主完成特定票據 (Ticket) 上的開發任務。
  - 負責執行終端機指令 (如安裝套件、跑單元測試) 以完成自身任務。

## 3. 在 Antigravity 中呼叫 Claude Code 的方法

在 Antigravity 中，我們可以直接透過 `run_command` 工具操作終端機來驅動 Claude Code。

### 方式 A：單次指令 (One-shot Task)
當 Antigravity 分派一個明確的任務時，使用一句話讓 Claude Code 完成並自動退出：
```bash
claude "幫我檢查與修復 src/auth.ts 中的 JWT 驗證邏輯問題"
```
*(如果環境變數抓不到 `claude`，可使用 `npx -y @anthropic-ai/claude-code "任務內容"` 替代)*

### 方式 B：指定上下文範圍
讓 Claude Code 聚焦在特定檔案上：
```bash
claude "實作 @src/components/LoginModule.tsx 裡面的 UI，根據 Tailwind 規範"
```

### 方式 C：搭配專案初始化
在讓 Claude Code 介入前，Antigravity 可以先幫忙建立好 `CLAUDE.md`，定義好專案規範，然後再呼叫它：
```bash
claude /init
```

## 4. 這個架構的優勢

1. **突破 Token 與 Context 限制**：Antigravity 可以保持宏觀的系統視野，而龐大的程式碼細節交給 Claude Code 處理，避免單一 Agent 記憶體超載。
2. **平行處理潛力**：若專案結構允許，Antigravity 理論上可以開啟多個終端機，平行呼叫多個 Claude Code CLI 同時處理不衝突的模組。
3. **優勢互補**：Antigravity 擅長操作各種工具 (瀏覽器、讀取網頁、架構規劃)，而 Claude CLI 在開發者本地終端機擁有極好的 Git 整合與程式碼推理能力。
